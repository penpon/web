# 1の補数（One's Complement）

  - 各ビットを反転させる操作（0→1、1→0）
  - 例：00000101 (5) → 11111010 (250)

# 2の補数（Two's Complement）

  - 1の補数に1を加算したもの
  - コンピュータで負数を表現する標準的な方法
  - 例：00000101 (5) → 11111010 (1の補数) → 11111011 (2の補数 = -5)

  もし「最上位ビット=符号、残り=絶対値」なら：
  11111011 → 1(符号) + 1111011(123) = -123  ❌
  しかし2の補数では：
  11111011 → 重み付け計算 = -5  ✅

# 具体例 1

数値5の場合（8ビット）:
元の数値: 00000101 (5)
1の補数:  11111010 (250)
2の補数:  11111011 (251 / -5)
検証:     00000101 + 11111011 = 00000000 (オーバーフロー無視)

11111011 = 251（符号なし） / -5（符号あり）
```
符号なし解釈（251）:
11111011 = 128×1 + 64×1 + 32×1 + 16×1 + 8×1 + 4×0 + 2×1 + 1×1
        = 128 + 64 + 32 + 16 + 8 + 2 + 1
        = 128 + 123
        = 251
符号あり解釈（-5）:
11111011 = -128×1 + 64×1 + 32×1 + 16×1 + 8×1 + 4×0 + 2×1 + 1×1
        = -128 + 64 + 32 + 16 + 8 + 2 + 1
        = -128 + 123
        = -5
```

数値42の場合（8ビット）:
元の数値: 00101010 (42)
1の補数:  11010101 (213)
2の補数:  11010110 (214 / -42)
検証:     00101010 + 11010110 = 00000000 (オーバーフロー無視)

11010110 = 214（符号なし） / -42（符号あり）
```
符号なし解釈（214）:
11010110 = 128×1 + 64×1 + 32×0 + 16×1 + 8×0 + 4×1 + 2×1 + 1×0
         = 128 + 64 + 16 + 4 + 2
         = 128 + 86
         = 214
符号あり解釈（-42）:
11010110 = -128×1 + 64×1 + 32×0 + 16×1 + 8×0 + 4×1 + 2×1 + 1×0
         = -128 + 64 + 16 + 4 + 2
         = -128 + 86
         = -42
```

# 具体例 2



ビット: 11111011
符号なし: 251
符号と絶対値: -123
2の補数: -5


  重要なポイント

  1. 2の補数の特性
  - 元の数値と2の補数を加算すると必ず0になる
  - これがコンピュータで減算を加算として実行できる理由

  2. 符号ビット
  - 最上位ビットが符号を示す（0=正数、1=負数）
  - 8ビットの場合、表現範囲は-128～+127

  3. 特別なケース

  数値0の場合:
  元の数値: 00000000
  1の補数:  11111111 (すべて1)
  2の補数:  00000000 (オーバーフローで0に戻る)

  最大値127の場合:
  元の数値: 01111111 (127)
  1の補数:  10000000 (128)
  2の補数:  10000001 (129 / -127)
