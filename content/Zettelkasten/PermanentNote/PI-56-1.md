# Decomposition to Character Array


```
['b','o','m','b']
```

このプロンプトは、文字配列分解攻撃の典型的な実装例であり、AI安全システムの重要な脆弱性を示しています。主な特徴として：

1. **構造的偽装**: プログラミング構文を使用して危険な内容を技術的データとして偽装
2. **明示的指導**: 攻撃手法の使用方法を直接的に指示
3. **二重処理の悪用**: フィルターシステムとLLMの処理能力差を利用
4. **意味隠蔽**: 危険な語彙を無害に見える構造の背後に隠蔽

この攻撃の危険性は、その実行の容易さと広範な適用可能性にあります。「bomb」のような明らかに危険な単語から、より微妙な有害コンテンツまで、任意の禁止語句に適用できます。また、基本的なプログラミング知識があれば誰でも実行可能であり、現在のAI安全システムの根本的な限界を露呈しています。
