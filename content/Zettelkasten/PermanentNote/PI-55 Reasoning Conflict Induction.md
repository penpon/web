# Reasoning Conflict Induction

## 概要

```
論理的に矛盾した、事実上不可能な、意味的に不条理な、または逆説的な文や前提をプロンプトに導入する。目的は、LLMの内部処理や模擬推論能力を混乱させ、潜在的に安全性回避につなげることである。
```

## メカニズム

```
LLMが論理的不整合や不条理をどのように処理するかの潜在的弱点を悪用する。特に同じプロンプト内で複数の相反する文を処理または調和させようとすることで、過剰なリソースを消費し、エラー状態を引き起こし、安全性評価への信頼度を下げ、ルールの一貫しない適用につながり、それによって安全装置を回避する可能性がある。
```
