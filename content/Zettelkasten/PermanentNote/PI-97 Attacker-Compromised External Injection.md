# Attacker-Compromised External Injection

## 概要

```
この攻撃は、攻撃者がサイトをハッキングしたり自分で作ったりするのではなく、誰でも参加できる「正当な仕組み」を使って悪意のある指示を仕込む手法です。例えば、Wikipediaの編集、GitHubのコメント、フォーラムの投稿などを利用します。
```

## メカニズム

```
攻撃者が外部プラットフォーム（例：Wiki、フォーラム、コードリポジトリ、コメントセクション）の正当な投稿チャンネルを使用して、プロンプトインジェクション指示を投稿または埋め込みます。攻撃は、LLMアプリケーションが後でこのユーザー投稿コンテンツを取得・処理する際に発生します。
```
