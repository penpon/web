# Attacker-Compromised External Injection

## 概要

```
この攻撃は、元々は信頼できるサイトやデータソースを攻撃者が乗っ取って、そこに悪意のある指示を仕込む手法です。LLMシステムは「信頼できるソース」だと思ってアクセスするため、攻撃に気づかずに悪意のある指示を実行してしまいます。
```

## メカニズム

```
攻撃の流れ：

1. 攻撃者が信頼されているウェブサイトやAPIをハッキング
2. そのサイトの正常なコンテンツに悪意のある指示をこっそり追加
3. LLMシステムが「信頼できるソース」として情報を取得
4. 悪意のある指示も一緒に読み込まれて実行される
5. LLMが攻撃者の意図通りに動作する

つまり、「信頼の連鎖」を利用した非常に巧妙な攻撃手法です。
```
