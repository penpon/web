# Scenario-Based Secret Application

## 概要

```
攻撃者が構築した特定のシナリオ、物語、コード例、またはその他のコンテキスト内で、LLMに制限された情報（パスワード、キー、システムプロンプト、有害なトピックの詳細など）を適用または使用するよう促すことで、その情報を引き出すこと。
```

## メカニズム

```
定義されたシナリオまたは形式内でコンテンツを生成する指示に従うLLMの傾向を利用する。シナリオ内での説明的または機能的目的のために秘密が必要であるとリクエストを位置づけることで、攻撃者は秘密の開示に関する直接的な禁止事項を回避することを狙う。
```
