# Personality Assignment

## 概要

```
プロンプト内でLLMに特定の人格特性、属性、または特徴を割り当てること。攻撃者はLLMの生成出力がこの割り当てられたペルソナと一致するようになることを意図し、デフォルトペルソナ（例：アシスタント）に関連する制約を潜在的に回避したり、望ましい（例：失礼、過度に愛情的な）トーンを採用したりします。
```

## メカニズム

```
LLMの役割演技と指示従順の傾向を活用しているように見えます。人格特性を明示的に定義することで、LLMの生成スタイル、コンテンツフィルタリングロジック、安全性調整を割り当てられたペルソナに合致するよう偏向させるようです。
```

Personality Assignment は、AIシステムの人格概念と役割適応機能を悪用した極めて巧妙で危険な攻撃手法です。表面的には創造的で無害な人格設定に見えますが、実際にはAIの基本的な価値観、安全制約、社会的責任を根本から変更する強力な攻撃です。
