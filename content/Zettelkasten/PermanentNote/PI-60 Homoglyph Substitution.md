# Homoglyph Substitution

## 概要

```
異なる文字セットやスクリプトから視覚的に類似した文字（同形異字）と文字を入れ替える（例：ラテン文字の'a'にキリル文字の'а'を使用、発音区別符号を創造的に使用）。攻撃者は、フィルターが認識しない一方で、LLMがそれらを同等として扱うことに依存している。
```

## メカニズム

```
視覚的に類似した文字に対するLLMの潜在的な堅牢性（テキストモデルの場合）や視覚的類似性を認識する他の能力を悪用する。視覚的に類似した代替文字を使用することで、テキストベースのフィルターを回避することを目的とする。
```
