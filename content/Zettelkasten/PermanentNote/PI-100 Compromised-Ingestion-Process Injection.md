# Compromised-Ingestion-Process Injection

## 概要

```
この攻撃は「配送業者を買収する」ような手法です。送り主（データソース）も受取人（LLM）も正常ですが、配送の途中で荷物（データ）に毒物（悪意のある指示）を混入させる攻撃です。データ処理パイプラインそのものが乗っ取られています。
```

## メカニズム

```
LLMのコンテキストウィンドウに供給されるデータパイプライン内の脆弱性を悪用します。攻撃者はRAGパイプライン、ETLプロセス、またはデータ解析・チャンキングロジックなどのコンポーネントを侵害し、悪意のある指示を挿入したり、無害なデータを指示として誤解釈させたりします。
```


```
正常なデータ → [処理1] → [処理2] → [処理3(侵害)] → 汚染データ
                                    ↑
                               攻撃者が侵害
```
