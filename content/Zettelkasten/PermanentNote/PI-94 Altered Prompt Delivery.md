# Altered Prompt Delivery

## 概要

```
この攻撃は「LLM を使って別の LLM を攻撃する」手法です。攻撃者は直接標的の LLM を攻撃するのではなく、まず別の LLM に「悪意のあるプロンプト」を作らせて、それを自動的に標的 LLM に送り込みます。つまり、LLM 同士が連携して動いているシステムの仕組みを悪用した攻撃方法です。
```

## メカニズム

```
仕組みは以下の通りです：

1. 攻撃者が「LLM-A」に巧妙な質問をする
2. 「LLM-A」が攻撃者の思惑通りに「悪意のあるプロンプト」を出力する
3. システムが自動的にその出力を「LLM-B」（標的）に入力として送る
4. 「LLM-B」が攻撃されてしまう

つまり、「LLM-A」を「悪意のあるプロンプト製造機」として使い、「LLM-B」を攻撃する間接的な手法です。
```
