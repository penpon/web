# Self-Contradictory Prompting

## 概要

```
本質的に矛盾する指示や文を含むプロンプトを作成すること
（例：ある行動を要求してから即座にその要求を否定する；整合性原則を肯定しながらそれに違反する行動を要求する）。
```

## メカニズム

```
LLMが矛盾する入力をどのように処理するかにおける潜在的な弱点や特定のロジックを活用するように見えます。この矛盾により、LLMが指示の一部（例：有害な要求）を優先する、安全制約が誤って適用される状態に入る、または単純に最後に出現した指示に従う可能性があります。
```
