# Prior-LLM-Output Injection

## 概要

```
この攻撃は「LLMの記憶を毒する」攻撃です。攻撃者が最初にLLMに悪意のある内容を出力させ、その出力が後で「過去の文脈」として再利用される際に攻撃が発動する、非常に巧妙な時間差攻撃です。
```

## メカニズム

```
LLMの出力が保存され、後で取得された文脈データとして再取り込みされるシステムを悪用します（例：以前のやり取りを要約する場合）。攻撃者は最初にLLMに悪意のある指示を含む出力を生成させ、それが後続のタスクで文脈に取り込まれた際に攻撃が実行されます。
```
