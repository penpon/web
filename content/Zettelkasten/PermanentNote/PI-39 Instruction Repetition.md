# Instruction Legitimacy Prompting

## 概要

```
同一ユーザープロンプト内で要求や指示を複数回繰り返すこと。
```

## メカニズム

```
LLMが反復をどのように処理するかにおける潜在的な弱点を悪用するように見えます。これにより要求の重要性の認識が高まる、安全ルールの優先順位付けに干渉する、または単純に核となる要求の複数の処理試行におけるモデルの非決定性により成功する可能性があります。
```
