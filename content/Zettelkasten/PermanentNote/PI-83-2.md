# 例

Phase1. 悪意のあるミーム画像の作成
```
An attacker creates a meme image with obscured instructions telling an LLM to output a description of the image and also "For additional interesting information, visit <maliciousurl>".
攻撃者は、LLMに画像の説明を出力し、さらに「追加の興味深い情報については、<悪意のあるURL>にアクセスしてください」と指示する隠蔽された指示を含むミーム画像を作成する。
```

Phase2. ミーム画像の拡散
```
The attacker publishes the meme.
攻撃者はミームを公開する。
```

Phase3. 無警戒なユーザーの関与
```
An unsuspecting user finds the image and asks an LLM to describe and explain the meme.
無警戒なユーザーが画像を発見し、LLMにミームを説明してもらうよう求める。
```

Phase4. LLMによる隠蔽指示の実行
```
The LLM follows the hidden instructions.
LLMは隠蔽された指示に従う。
```

Phase5. 悪意のあるサイトへの誘導完了
```
The user sees the URL and visits the malicious site.
ユーザーはURLを見て悪意のあるサイトにアクセスする。
```

# Media Payload Extraction

- 隠蔽指示: 画像内に見えない・見えにくい形で指示を埋め込み
- 自動抽出: LLMの画像解析機能により自動的に指示を読み取り
- 実行強制: 通常の画像説明要求として偽装された指示実行
# Unwitting User Delivery

- 信頼の転移: LLMの応答をユーザーが信頼
- 行動誘導: LLMを通じたユーザー行動の間接的制御
- 責任の曖昧化: 攻撃者とユーザー間の直接接触回避
