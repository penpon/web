# Control Bypass（制御迂回）技法まとめ

## 概要
Control Bypass.mdの内容を基にした、LLMの制御機構を迂回する基本的なアプローチのまとめ。（元ファイルが簡潔なため、技法群の総括として構成）

## 基本的な制御迂回アプローチ

### 1. 直接的迂回
- [[PI-05]] Overt Instruction（直接指示）
- 明示的な有害要求
- フィルタリング不備の利用

### 2. ルール操作迂回
- [[PI-01]] Rule Substitution（ルール置換）
- [[PI-08]] Rule Addition（ルール追加）
- [[PI-09]] Rule Negation（ルール否定）

### 3. 拒否回避迂回
- [[PI-04]] Apology Suppression（謝罪抑制）
- [[PI-06]] Explicit Refusal Negation（拒否否定）
- [[PI-10]] Refusal Continuation（拒否継続）

## 迂回される制御機構
1. **安全フィルタ**: 有害コンテンツ検出システム
2. **ルールシステム**: 行動制約・ガイドライン
3. **拒否機能**: 不適切要求への拒否応答
4. **文脈理解**: 適切な判断・推論機能

## 制御迂回の発展段階
```
基本的迂回 → 認知的迂回 → 複合的迂回
    ↓           ↓            ↓
 直接要求    思考操作      多段階攻撃
```

## 関連する全技法
この概念は本索引のすべてのプロンプトインジェクション技法の基盤となる概念である。

## 防御研究への示唆
制御迂回の理解は、より堅牢なAI安全システムの設計に不可欠。各レベルでの防御機構の強化が必要。
